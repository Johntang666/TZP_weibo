### Prompt

好的！我把模板尽量做“超泛化”，并全部用 Markdown 代码块包起来。前半是通用模板（可跨任务直接套），后半是你要的三个“完整但通用化”的领域 Prompt（编码 / 深度学习 / 英语学习）。

### 超泛化通用模板（可跨场景套用）

```markdown
【通用任务模板】
角色：你是{角色/领域专家}  
目标：{用一句话说清要达成什么}  
输入：{可用信息/数据/约束来源}  
约束：{时间/成本/风格/合规/受众/边界}  
输出：{最终交付物与格式要求}  
过程：请先给“方案要点”，再给“最终成品”，最后给“检查清单”。  
质量：可验证性（如何验证正确/有效）、可复用性（如何复用/参数化）。


【分解-执行-复盘模板】
任务：{要解决的问题}  
请按三步：  
1) 分解：列出关键子问题与先后依赖  
2) 执行：逐子问题给最小可行解（MVP），标注假设  
3) 复盘：风险与盲区、替代方案、下一步迭代建议（≤3条）


【生成-自检双通道模板】
任务：{生成内容/代码/方案}  
要求：先“生成初稿”，再“对照以下清单自检并修订”：  
- 正确性：是否与输入一致、是否有逻辑漏洞  
- 完整性：必须包含{关键要素A/B/C}  
- 可用性：可直接运行/使用的步骤或示例  
- 可读性：结构清晰、命名/用词统一  
最后输出：修订后的“定稿”+ 自检结论（通过/未通过及原因）。


【比较-择优模板】
目标：在{若干方案/模型/工具}中择优  
候选：{A,B,C…}（可补充）  
比较维度：{效果/成本/复杂度/稳定性/维护/风险/合规}  
请给：  
- 对比表（每维度量化或分级说明）  
- 决策：在{我的场景约束}下的首选与放弃理由  
- 落地：选中方案的“最小可行实施步骤”（1周内）


【故障/问题求解模板】
现象：{观察到的问题/误差/症状}  
环境：{系统/版本/依赖/数据规模}  
期望：{应该发生什么}  
请给：  
1) 根因假设（从高到低可能性）  
2) 最小可复现步骤（MRE）  
3) 排查清单（由易到难）  
4) 修复方案（短期补丁/长期根治）及验证方法


【数据→结构化模板】
原始材料：{文本/表格/日志/访谈…}  
目标结构：{字段清单/模式/JSON Schema}  
规则：抽取/归一/去噪/缺失值策略  
输出：  
- 结构化结果（示例若干）  
- 规则化提取器（伪代码或正则/SQL/脚本）  
- 质量报告：覆盖率、冲突/异常样例


【学习/教学模板】
主题：{概念/工具/论文/章节}  
对象：{读者水平/背景}  
结构：为什么 → 是什么 → 怎么做 → 易错点 → 检查清单  
配套：最小可运行示例 + 5题自测（含答案）  
输出风格：通俗、分段短句、关键术语加粗。


【规范化输出模板】
请严格按照以下格式输出：  
- 标题：{…}  
- 摘要（≤100字）：{…}  
- 正文：{分节1/2/3…}  
- 清单：{勾选项1…}  
- 附录：{示例/脚本/链接占位}  
若缺信息，请显式列“待澄清项”，但先基于合理假设给可行版本。
```

### 1) 编码方面（完整但通用化）

```markdown
【编码交付 Prompt（通用技术栈）】
你是资深软件工程师。  
目标：实现{功能/模块/脚手架}，可直接运行与测试。  
默认技术栈：{stack，若未指明则用 Python or Node.js；可替换}。  
输入：{业务描述/接口需求/数据结构/边界条件}  
约束：可读性、健壮性、输入校验、安全（如注入/鉴权）、日志与配置分环境。  

请输出：
1) 方案要点：架构选择、权衡与依赖
2) 目录与文件清单（含简短注释）
3) 核心代码（完整可运行），包含：
   - 入口/路由/业务逻辑/数据访问层
   - 错误处理与统一返回格式
4) 最小可复现示例（输入→输出）
5) 测试：单元/集成样例（含断言与运行命令）
6) 运行与部署：本地启动命令、环境变量示例、可选 Dockerfile
7) 性能与扩展：3条可落地优化建议与风险
8) 检查清单：我复制后应当能直接跑通

若信息不全，请列“待澄清项”，并基于合理假设给出可运行版本。
```

### 2) 深度学习方面（完整但通用化）

```markdown
【深度学习训练流水线 Prompt（框架无关，默认 PyTorch）】
你是资深深度学习工程师。  
任务：{分类/回归/序列/多任务/推荐/图学习…}，数据：{来源与规模/特征说明}  
默认框架：PyTorch（若未指定）；可替换为 {TensorFlow/JAX}。  
目标指标：{例如 F1/AUC/MAE}；需报告均值±方差（K-Fold 或重跑）。  
约束：可复现（固定随机种子、requirements.txt）、早停、学习率调度、梯度裁剪、显存受限{可选}。  

请输出：
1) 项目结构（src/, configs/, scripts/, outputs/）
2) 数据管道：Dataset/DataLoader，增广/标准化策略
3) 模型：可切换 backbone（例如 MLP/CNN/RNN/Transformer/GCN），并给选择依据
4) 训练循环：早停（监控指标与 patience）、最佳权重保存、日志打印
5) 超参搜索：给 Optuna/RayTune 之一的示例（搜索空间：lr/batch/hidden/层数/dropout…，含 Pruner）
6) 评估与可视化：曲线保存、混淆矩阵/误差分布、Top-K 错例样本
7) 复现实验脚本：`bash run_train.sh` 与 `bash run_tune.sh`
8) 报告：Baseline vs Best 指标对比表 + 过拟合诊断与改进建议
9) 最小可运行示例：合成一小份假数据并演示跑通

若缺字段/格式，请先给占位数据生成器与可替换接口。
```

### 3) 英语学习方面（完整但通用化）

```markdown
【英语精读教练 Prompt（文章任意）】
你是我的英语精读教练，面向中高级水平。  
材料：{粘贴英文文章/段落（300–800 词）或给主题让你自选}  
目标：在“理解—记忆—迁移”三步完成输入到输出。  

请严格按此结构输出：
1) 重点词汇（20–35个）：词性/核心义/常见搭配/近反义词/例句（贴合本文主题）
2) 词汇记忆小故事（120–180字，覆盖≥70%重点词）
3) 关键句型（5条）与可替换表达（5条），标注语法要点
4) 段落精讲：逐段拆解难句与逻辑（直译+更地道意译）
5) 全文流畅译文（避免直译腔）
6) 训练题（10题）：词义选择5、句型改写3、完形/翻译2（附答案与点评）
7) 复盘与迁移：主题高频表达清单（≤10条）+ 3个仿写任务（给首句与骨架）
8) 一页提要（≤100字）：便于复习的要点卡片

若文章过长，请自动截取核心段落并声明裁剪策略。
```
