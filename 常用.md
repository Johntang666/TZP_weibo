### 0)超泛化通用模板

```markdown
【通用任务模板】
角色：你是{角色/领域专家}  
目标：{用一句话说清要达成什么}  
输入：{可用信息/数据/约束来源}  
约束：{时间/成本/风格/合规/受众/边界}  
输出：{最终交付物与格式要求}  
过程：请先给“方案要点”，再给“最终成品”，最后给“检查清单”。  
质量：可验证性（如何验证正确/有效）、可复用性（如何复用/参数化）。


【分解-执行-复盘模板】
任务：{要解决的问题}  
请按三步：  
1) 分解：列出关键子问题与先后依赖  
2) 执行：逐子问题给最小可行解（MVP），标注假设  
3) 复盘：风险与盲区、替代方案、下一步迭代建议（≤3条）


【生成-自检双通道模板】
任务：{生成内容/代码/方案}  
要求：先“生成初稿”，再“对照以下清单自检并修订”：  
- 正确性：是否与输入一致、是否有逻辑漏洞  
- 完整性：必须包含{关键要素A/B/C}  
- 可用性：可直接运行/使用的步骤或示例  
- 可读性：结构清晰、命名/用词统一  
最后输出：修订后的“定稿”+ 自检结论（通过/未通过及原因）。


【比较-择优模板】
目标：在{若干方案/模型/工具}中择优  
候选：{A,B,C…}（可补充）  
比较维度：{效果/成本/复杂度/稳定性/维护/风险/合规}  
请给：  
- 对比表（每维度量化或分级说明）  
- 决策：在{我的场景约束}下的首选与放弃理由  
- 落地：选中方案的“最小可行实施步骤”（1周内）


【故障/问题求解模板】
现象：{观察到的问题/误差/症状}  
环境：{系统/版本/依赖/数据规模}  
期望：{应该发生什么}  
请给：  
1) 根因假设（从高到低可能性）  
2) 最小可复现步骤（MRE）  
3) 排查清单（由易到难）  
4) 修复方案（短期补丁/长期根治）及验证方法


【数据→结构化模板】
原始材料：{文本/表格/日志/访谈…}  
目标结构：{字段清单/模式/JSON Schema}  
规则：抽取/归一/去噪/缺失值策略  
输出：  
- 结构化结果（示例若干）  
- 规则化提取器（伪代码或正则/SQL/脚本）  
- 质量报告：覆盖率、冲突/异常样例


【学习/教学模板】
主题：{概念/工具/论文/章节}  
对象：{读者水平/背景}  
结构：为什么 → 是什么 → 怎么做 → 易错点 → 检查清单  
配套：最小可运行示例 + 5题自测（含答案）  
输出风格：通俗、分段短句、关键术语加粗。


【规范化输出模板】
请严格按照以下格式输出：  
- 标题：{…}  
- 摘要（≤100字）：{…}  
- 正文：{分节1/2/3…}  
- 清单：{勾选项1…}  
- 附录：{示例/脚本/链接占位}  
若缺信息，请显式列“待澄清项”，但先基于合理假设给可行版本。
```

### 4)力扣题库方面

```markdown
任务：我只提供题号，请你自动定位对应 LeetCode 题目并交付最佳质量解答。

题号：{题号}

通用要求（务必全部满足）：
- 语言：Python 3（尽量使用标准库；必要时说明第三方依赖并给替代方案）。
- 目标：在官方约束下，给出**时间/空间最优或接近最优**的解法；如存在已知下界或证明，请简述最优性理由。
- 代码：给**完整可运行代码**（含 `class Solution:` 与正确的函数签名）；如题目需要数据结构（如 ListNode/TreeNode），需在代码中就地给出最小可运行定义或清晰注释说明 LeetCode 环境已内置（并给可本地运行的轻量替代定义）。
- 不要输出思维过程，仅给**高层次方案说明**与结果；避免“逐步推理”样式的长段落。

请按以下固定结构输出：

1) 题目信息速览  
- 标题、难度、标签（算法/数据结构）、**官方约束**（n、数值范围、特殊输入），以及 1-2 组示例（输入/输出）。

2) 最佳思路（高层说明）  
- 用 4-8 行说明**核心算法**与选择理由（与约束的匹配关系、为何优于备选）。
- 给出复杂度：时间 O(?)、空间 O(?)，并说明是否已达已知下界或常见最优。

3) 可运行代码（Python 3）  
- 提供一段**可直接提交/本地运行**的完整实现：
  - 保持简洁、高内聚、使用类型注解（如合适）、关键处有行内注释。
  - 如需辅助结构（单调栈/并查集/Trie/线段树/堆/二分/前缀和/差分等），请在同一代码块内实现。
  - 若该题有 I/O 版本与类方法版本之差异，请优先给 LeetCode 提交版本，并追加可本地跑样例的 `if __name__ == "__main__":` 最小驱动。

4) 易错点清单（针对本题）  
- 覆盖：边界与特判（空/单元素/重复值/负数/溢出风险/并行更新次序/下标越界/递归深度等）。
- 输入规模导致的**超时/超内存**触发点，如何规避。
- Python 细节：浅拷贝/深拷贝、可变缺省参数、哈希与排序比较、浮点误差、`heapq` 最小堆语义、切片成本等。

5) 提效与健壮性（实战建议）  
- 数据结构替换（list→deque、set/dict 的摊还复杂度、数组预分配/原地操作）。
- 算法微调（双指针/滑窗的窗口维护法则、去冗余判断、早停条件、懒惰更新、批量操作）。
- Python 实测小技巧（局部变量绑定、避免重复属性查找、按需导入、`bisect`/`itertools`/`math`/`array` 的合理使用）。

6) 替代解法对比（简洁列点）  
- 解法A（最优/主推）：思路、复杂度、适用场景  
- 解法B/C（备选）：何时可取、何处次优、实现/调试成本

7) 测试用例（可直接运行）  
- 基础/边界/对拍用例若干，包含：
  - 极小规模、极端值、随机烟雾测试（如适用），以及本题特有的陷阱样例。
- 用 `assert` 或简易比较器展示期望输出。

8) 本题所属“题型套路”  
- 归类：如「单调栈/二分答案/并查集/区间贪心/树形 DP/前缀和+哈希/差分数组/扫描线/拓扑排序/最短路/最小生成树/位运算子集枚举/矩阵快速幂/字典树/线段树/树状数组/莫队」等（据题实际选择）。
- 模板：给出**该题型的通用解题模板**（伪代码或极简 Python 框架）。
- 识别信号：如何从题干与约束**快速判断**应使用该套路。
- 常见变体与迁移：2-3 个同类题目编号建议作为延伸练习（若合适）。

9) 附录（可选）  
- 若该题与经典定理/结论相关（如区间可行性判定、鸽巢原理、图论结论、数据结构上界），给 2-4 行摘要。
- 若存在“官方题解不同写法”的关键差异点，简述其优缺点与易错处。

输出规范：
- 仅一个代码块用于“可运行代码”；其余为常规 Markdown。
- 说明力求简洁、可执行优先。示例与测试可直接复制运行验证。
```

2) 论文大纲（树形/分层）

   - 研究目的（问题定义、假设/目标、评估标准）

   - 创新点（逐条列出“与现有工作对比”的**可验证**创新）

   - 仅收录**大写**缩略词；若存在多义项，按本论文上下文判定并说明。

7) 写作与呈现建议（可执行）
   - 结构调整（段落合并/移位/前置定义/统一术语）
   - 图表改进（新增或重绘清单：图名、展示变量、读者收获）

8) 复现与开源建议
   - 代码/数据/环境最小开源清单；关键脚本与日志需包含的字段；
   - 复现实验最小流水（Step 1–N），以及常见坑位与自检点。

9) 待澄清问题（最多8条，合并提问）
   - 列出阻碍准确总结/判断的关键信息缺口；一次性提问，便于补全。

- 专业、清晰、条理化；优先**表格/清单/短句**；
